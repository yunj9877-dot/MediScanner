# MediScanner 검색 시스템 (SEARCH_RETRIEVAL.md)

## 1. 서비스 컨셉: 왜 검색 정확도가 중요한가

### 1.1 메디스캐너란?

**메디스캐너(MediScanner)**는 **개인 맞춤형 의료 질의 서비스**입니다.

일반적인 의료 챗봇과 다른 점:

| 일반 의료 챗봇 | 메디스캐너 |
|--------------|-----------|
| "고혈압 치료법은?" → 일반적인 답변 | "고혈압 치료법은?" → **이 사용자의 기저질환과 복용약을 고려한** 맞춤 답변 |
| 모든 사람에게 같은 답변 | 사용자마다 다른 답변 |

### 1.2 개인 맞춤 답변의 예시

**사용자 프로필**:
- 기저질환: 고혈압, 당뇨
- 복용약: 아스피린, 메트포르민

**질문**: "감기약 먹어도 돼?"

**일반 챗봇 답변**:
> "감기 증상에 따라 해열제, 진통제 등을 복용할 수 있습니다."

**메디스캐너 답변**:
> "① 고혈압: 슈도에페드린 성분이 들어간 감기약은 혈압을 올릴 수 있어 피하세요.
> ② 당뇨: 당분이 많은 시럽형 감기약보다 정제형을 선택하세요.
> 💊 복용약 주의: 아스피린과 일부 감기약(이부프로펜 등)은 함께 복용 시 위장 출혈 위험이 있습니다.
> ※ 참고용 정보이니, 정확한 진단은 의료 전문가와 상담하세요."

### 1.3 왜 검색 정확도가 핵심인가

개인 맞춤 답변을 위해서는 **정확한 의료 문서 검색**이 필수입니다.

```
사용자 질문 + 프로필
        ↓
    [검색 시스템] ← 이 단계가 부정확하면?
        ↓
관련 의료 문서 5개 선택
        ↓
    [GPT-4o-mini]
        ↓
    맞춤형 답변
```

**검색이 실패하면**:
- 엉뚱한 문서가 선택됨
- GPT가 잘못된 정보로 답변 생성
- **위험한 의료 정보 제공 가능**

**따라서**: 메디스캐너에서 검색 정확도는 **서비스 품질의 핵심**입니다.

## 3. 세 가지 검색 방법: 쉬운 설명

### 3.1 BM25 (키워드 검색)

**한마디로**: **"글자가 똑같으면 찾는다"**

**비유**: 도서관에서 책 찾기
```
"고혈압" 책 찾아주세요
        ↓
사서가 책 제목/내용에서 "고혈압" 글자를 찾음
        ↓
"고혈압"이 많이 나오는 책일수록 상위 추천
```

**예시**:
| 질문 | 문서 | 결과 |
|------|------|:----:|
| "아스피린 부작용" | "아스피린은 위장에..." | ✅ 찾음 |
| "두통약 부작용" | "아스피린은 위장에..." | ❌ 못 찾음 (두통약≠아스피린) |
| "고혈압" (한국어) | "Hypertension is..." (영어) | ❌ 못 찾음 (언어가 다름) |

**장점**: 정확한 단어(약 이름, 음식 이름) 찾기에 강함
**단점**: 같은 뜻 다른 말, 다른 언어는 못 찾음

---

### 3.2 시맨틱 검색 (의미 검색)

**한마디로**: **"뜻이 비슷하면 찾는다"**

**비유**: 똑똑한 사서에게 물어보기
```
"머리 아플 때 먹는 약" 찾아주세요
        ↓
사서: "아, 진통제 말씀하시는 거죠?"
        ↓
"아스피린", "타이레놀" 관련 책 추천
```

**예시**:
| 질문 | 문서 | 결과 |
|------|------|:----:|
| "두통약 부작용" | "aspirin side effects" | ✅ 찾음 (뜻이 같으니까) |
| "고혈압" (한국어) | "Hypertension is..." (영어) | ✅ 찾음 (의미가 같으니까) |
| "아스피린" | "아세틸살리실산 부작용" | ❌ 놓칠 수 있음 (고유명사 약함) |

**장점**: 다른 언어, 다른 표현도 의미로 찾음
**단점**: 정확한 고유명사(약 이름 등)는 놓칠 수 있음

---

### 3.3 RRF 하이브리드 (둘 다 합치기)

**한마디로**: **"두 방법 결과를 합쳐서 최고를 뽑는다"**

**비유**: 두 사서의 추천을 종합하기
```
질문: "혈압약 먹는데 자몽 먹어도 돼?"

[키워드 사서]              [의미 사서]
1위: 자몽 주의사항         1위: 약물 상호작용 
2위: 약물 상호작용         2위: 혈압약 복용법
3위: 과일 영양소          3위: 식이 주의사항

        ↓ 둘 다 추천한 건? ↓

[최종 결과]
1위: 약물 상호작용 ← 둘 다 추천했으니 이게 정답!
```

**핵심 아이디어**: 두 사서가 **모두 상위권에 올린 문서**가 가장 좋은 문서!

---

### 3.4 세 방법 비교 요약

| 방법 | 원리 | 잘 찾는 것 | 못 찾는 것 |
|------|------|-----------|-----------|
| **BM25** | 글자 매칭 | "아스피린", "자몽" | "두통약"→"아스피린" |
| **시맨틱** | 의미 매칭 | "고혈압"→"hypertension" | 정확한 약 이름 |
| **RRF** | 둘 다 합침 | 둘의 장점 결합 | 둘 다 못 찾으면 실패 |

## 4. 세 가지 검색 방법: 전문 설명

### 4.1 BM25 (Best Match 25)

**정의**: 단어의 출현 빈도(TF)와 희소성(IDF)을 기반으로 문서 관련성을 점수화하는 확률적 검색 알고리즘 [Robertson & Zaragoza, 2009]

**수식**:
```
score(D, Q) = Σ IDF(qi) × [ f(qi, D) × (k1 + 1) ] / [ f(qi, D) + k1 × (1 - b + b × |D|/avgdl) ]

- Q: 질문 (쿼리)
- D: 문서
- qi: 질문의 i번째 단어
- f(qi, D): 단어 qi가 문서 D에 나타난 빈도
- |D|: 문서 길이
- avgdl: 전체 문서의 평균 길이
- k1: 빈도 포화 상수 (일반적으로 1.2~2.0)
- b: 문서 길이 정규화 파라미터 (일반적으로 0.75)
- IDF(qi): 역문서 빈도 = log((N - n(qi) + 0.5) / (n(qi) + 0.5) + 1)
  - N: 전체 문서 수
  - n(qi): 단어 qi를 포함하는 문서 수
```

**MediScanner 적용**:
- 토크나이징: 한국어는 2글자 이상 단위로 분리
- 라이브러리: `rank_bm25.BM25Okapi`
- 파라미터: k1=1.5, b=0.75 (기본값)

**특성**:
| 장점 | 단점 |
|------|------|
| 정확한 키워드 매칭 (약물명, 고유명사) | 동의어 인식 불가 |
| 계산 속도 빠름 (O(n) 검색) | 언어 간 검색 불가 |
| 해석 가능성 높음 | 의미적 유사성 무시 |

**논문 레퍼런스**: Robertson, S. & Zaragoza, H. (2009). "The Probabilistic Relevance Framework: BM25 and Beyond." Foundations and Trends in Information Retrieval, 3(4), 333-389.

---

### 4.2 시맨틱 검색 (Dense Retrieval)

**정의**: 텍스트를 고차원 벡터(임베딩)로 변환하여 벡터 공간에서의 유사도로 관련성을 측정하는 검색 방법 [Karpukhin et al., 2020]

**작동 원리**:
```
[문서 저장 (오프라인)]
문서 텍스트 → 임베딩 모델 → 1024차원 벡터 → ChromaDB 저장

[검색 (온라인)]
질문 텍스트 → 임베딩 모델 → 1024차원 벡터
        ↓
ChromaDB에서 코사인 유사도 계산
        ↓
가장 유사한 벡터의 문서 반환
```

**코사인 유사도**:
```
cosine_similarity(A, B) = (A · B) / (||A|| × ||B||)

- A, B: 두 벡터
- A · B: 내적 (dot product)
- ||A||: 벡터 A의 크기 (L2 norm)
- 결과: -1 ~ 1 (1에 가까울수록 유사)
```

**MediScanner 적용**:
- 임베딩 모델: `intfloat/multilingual-e5-large` (1024차원)
- 벡터DB: ChromaDB
- E5 Prefix 규칙:
  - 문서 저장 시: `"passage: " + 문서`
  - 검색 쿼리 시: `"query: " + 질문`

**Cross-lingual Retrieval**:
```
multilingual-e5 모델의 특성:
- 100개 이상 언어 지원
- 같은 의미의 텍스트는 언어가 달라도 벡터 공간에서 가까이 위치

예시:
"고혈압"     → [0.23, -0.15, 0.82, ...]
"hypertension" → [0.25, -0.12, 0.79, ...]
  → 코사인 유사도 높음 → 검색 성공!
```

**특성**:
| 장점 | 단점 |
|------|------|
| 의미적 유사도 파악 | 계산 비용 높음 (GPU 권장) |
| Cross-lingual 검색 가능 | 고유명사/희귀 단어에 약함 |
| 동의어, 유사 표현 인식 | "시맨틱 유사 함정" (관련 없는데 유사) |

**논문 레퍼런스**: 
- Wang, L. et al. (2024). "Multilingual E5 Text Embeddings: A Technical Report." arXiv:2402.05672
- Reimers, N. & Gurevych, I. (2019). "Sentence-BERT: Sentence Embeddings using Siamese BERT-Networks." EMNLP 2019

### 4.3 RRF (Reciprocal Rank Fusion)

**정의**: 여러 검색 방법의 결과를 순위 기반으로 융합하는 알고리즘. 점수의 스케일이 다른 검색 방법들을 비교 가능하게 통합 [Cormack et al., 2009]

**RRF 수식**:
```
RRF_score(d) = Σ 1 / (k + rank_i(d))

- d: 문서
- k: 상수 (일반적으로 60)
- rank_i(d): i번째 검색 방법에서 문서 d의 순위 (1부터 시작)
- Σ: 모든 검색 방법에 대해 합산
```

**예시 계산**:
```
질문: "혈압약 먹는데 자몽 먹어도 돼?"

문서 A:
- 시맨틱 검색 순위: 1위 → 1/(60+1) = 0.0164
- BM25 검색 순위:   2위 → 1/(60+2) = 0.0161
- RRF 점수: 0.0164 + 0.0161 = 0.0325

문서 B:
- 시맨틱 검색 순위: 3위 → 1/(60+3) = 0.0159
- BM25 검색 순위:   1위 → 1/(60+1) = 0.0164
- RRF 점수: 0.0159 + 0.0164 = 0.0323

결과: 문서 A(0.0325) > 문서 B(0.0323) → 문서 A가 1위
```

**MediScanner 적용**:
```python
def rrf_fusion(semantic_docs, bm25_docs, k=60):
    rrf_scores = {}
    
    # 시맨틱 검색 결과 순위 반영
    for rank, doc in enumerate(semantic_docs):
        doc_id = doc["id"]
        rrf_scores[doc_id] = rrf_scores.get(doc_id, 0) + 1.0 / (k + rank + 1)
    
    # BM25 검색 결과 순위 반영
    for rank, doc in enumerate(bm25_docs):
        doc_id = doc["id"]
        rrf_scores[doc_id] = rrf_scores.get(doc_id, 0) + 1.0 / (k + rank + 1)
    
    # 점수 기준 정렬
    sorted_docs = sorted(rrf_scores.items(), key=lambda x: x[1], reverse=True)
    return sorted_docs
```

**핵심 아이디어**:
1. **점수가 아닌 순위 사용**: 서로 다른 스케일의 점수를 비교할 필요 없음
2. **상호 보완**: 한 방법에서 낮은 순위여도 다른 방법에서 높으면 최종 순위 상승
3. **k=60**: 상위 문서에 더 큰 가중치 부여 (논문 권장값)

**특성**:
| 장점 | 단점 |
|------|------|
| 두 검색 방법의 장점 결합 | 둘 다 실패하면 보완 불가 |
| 점수 정규화 불필요 | 추가 계산 비용 (미미함) |
| 튜닝 파라미터 적음 (k만 조절) | |

**논문 레퍼런스**: Cormack, G. V., Clarke, C. L. A., & Büttcher, S. (2009). "Reciprocal Rank Fusion outperforms Condorcet and Individual Rank Learning Methods." SIGIR 2009

| RRF k값 | 60 |
| GPU | RTX 4070 (CUDA) |

### 5.2 평가 방법

**Hit Rate@K 측정**:
- 질문에 대해 상위 K개 검색 결과 중 **정답 키워드가 포함된 문서가 있으면 Hit**
- Top-3, Top-5, Top-10 각각 측정

**평가 기준**:
```python
def check_hit(retrieved_docs, answer_keywords):
    """
    retrieved_docs: 검색된 문서 리스트
    answer_keywords: 정답에 포함되어야 할 키워드 리스트
    
    키워드 중 하나라도 문서에 포함되면 Hit
    """
    for doc in retrieved_docs:
        for keyword in answer_keywords:
            if keyword.lower() in doc.lower():
                return True
    return False
```

- **더 어려운 질문으로 재평가 필요**

### 5.4 실험 2: 어려운 질문 30개 (실제 사용자 스타일)

**질문 설계 원칙**:
- 실제 고령자가 할 만한 자연스러운 표현
- 추론이 필요한 간접적 표현
- 증상→질병, 생활습관→영향 관계 파악 필요

**질문 예시 (카테고리별)**:

| 카테고리 | 질문 예시 | 난이도 요인 |
|---------|----------|-----------|
| 증상-질병 | "머리가 어지러운데 이러면 고혈압에 안좋아?" | 증상↔질병 추론 필요 |
| 생활습관-영향 | "술을 많이 먹으면 골다공증에는 어떤 영향이 있어?" | 인과관계 파악 필요 |
| 음식-약물 | "혈압약 먹는데 자몽 먹으면 안 된다던데 왜야?" | 상호작용 지식 필요 |
| 복합질환 | "고혈압이랑 당뇨 같이 있으면 더 위험해?" | 다중 질환 관계 |
| 약물부작용 | "수면제 오래 먹으면 치매 걸려?" | 장기 복용 영향 |
| 예방관리 | "치매 걸리지 않으려면 뭘 해야 해?" | 예방법 지식 필요 |

**결과**:

| 검색 방식 | Top-3 | Top-5 | Top-10 |
|----------|:-----:|:-----:|:------:|
| 시맨틱만 | 90.0% | 90.0% | 93.3% |
| BM25만 | 80.0% | 86.7% | 96.7% |
| **RRF 하이브리드** | **86.7%** | **93.3%** | **93.3%** |

**RRF가 효과적인 경우 (상호 보완)**:

| "나이 들면 왜 자꾸 넘어져? 어떻게 예방해?" | ✅ | ❌ | ✅ | 시맨틱이 "낙상" 의미 파악 |

**RRF로도 해결 못한 경우 (둘 다 실패)**:

| "소변볼 때 찔끔찔끔 나오고 시원하지 않은데 왜 그래?" | ❌ | ❌ | ❌ | "찔끔찔끔"→"배뇨장애" 구어체 매핑 실패 |

## 6. 결과 분석

### 6.1 시맨틱(90%) > BM25(86.7%)인 이유

**핵심 원인: Cross-lingual 상황**

```
현재 데이터 구조:
- 질문: 한국어
- 문서: 영어 (CDC, WHO, Medscape)

BM25 한계:
질문: "고혈압 치료 방법은?"
  → 토큰: ["고혈압", "치료", "방법"]
  → 영어 문서에서 "고혈압" 검색
  → ❌ 매칭 실패 (영어 문서에 "고혈압" 단어 없음)

시맨틱 강점:
질문: "고혈압 치료 방법은?"
  → 벡터: [0.23, -0.15, 0.82, ...]
  → 영어 문서 "hypertension treatment" 벡터와 유사도 계산
  → ✅ 매칭 성공 (의미가 같으므로 벡터가 가까움)
```

**결론**: 한국어 질문 ↔ 영어 문서 환경에서는 시맨틱 검색이 유리

---

### 6.2 음식-약물에서 BM25(100%) > 시맨틱(50%)인 이유

**해당 질문**:
- "피 묽게 하는 약 먹는데 시금치 많이 먹어도 돼?"

**분석**:
```
시맨틱 실패 원인:
"피 묽게 하는 약" ← 이 표현이 "warfarin"과 의미적으로 직접 연결 안 됨
  → 임베딩 벡터가 멀리 위치
  → 관련 문서 검색 실패

BM25 성공 원인:
"시금치" 키워드가 문서에 직접 등장
  → 키워드 매칭으로 "vitamin K", "spinach" 문서 검색 성공
```

**일반화**: 구체적인 고유명사(약물명, 음식명)는 BM25가 강함

---

### 6.3 증상-질병에서 둘 다 60%로 낮은 이유

**실패한 질문 예시**:
```
질문: "요즘 자꾸 목이 마르고 화장실을 자주 가는데 무슨 병이야?"

사용자 표현      →    의학 용어        →    영어 문서
─────────────────────────────────────────────────────
"목이 마르다"    →    "다음증, 갈증"   →    "polydipsia"
"화장실 자주"    →    "빈뇨, 다뇨"     →    "polyuria"
```

**이중 장벽**:
1. **언어 장벽**: 한국어 질문 ↔ 영어 문서
2. **용어 장벽**: 일상 구어체 ↔ 의학 전문 용어

**문제점**:
- multilingual-e5는 일반 다국어 모델 (의료 특화 아님)
- "목이 마르다" → "polydipsia" 연결은 의료 도메인 지식 필요
- 현재 모델로는 추론 한계

**개선 방안**:
- 한국어 QA 데이터 추가로 직접적인 증상-질병 매핑 강화
- 또는 의료 특화 임베딩 모델(MedCPT 등) 검토

---

### 6.4 RRF 효과가 +3.3%p인 이유

**상호 보완 분석**:

| 케이스 | 시맨틱 | BM25 | RRF | 발생 빈도 |
|--------|:------:|:----:|:---:|:---------:|
| 둘 다 성공 | ✅ | ✅ | ✅ | 24/30 (80%) |
| 시맨틱만 성공 | ✅ | ❌ | ✅ | 2/30 (6.7%) |
| BM25만 성공 | ❌ | ✅ | ✅ | 1/30 (3.3%) |
| 둘 다 실패 | ❌ | ❌ | ❌ | 2/30 (6.7%) |
| 충돌 (드묾) | - | - | - | 1/30 (3.3%) |

**계산**:
- 시맨틱 단독: 24 + 2 = 26/30 = 86.7%... 실제는 27/30 = 90%
- RRF: 24 + 2 + 1 = 27/30 = 90%... 실제는 28/30 = 93.3%
- 상호 보완 효과: +3.3%p

**결론**: RRF는 효과가 있지만, **둘 다 실패하는 케이스(6.7%)는 해결 불가**

## 7. 향후 실험 계획

### 7.1 한국어 QA 데이터 임베딩 및 테스트

**데이터**: 라벨링데이터 (~25,000개 한국어 QA)

**예상 효과**:
- 한국어 질문 ↔ 한국어 문서 직접 매칭
- 증상-질병 카테고리 정확도 향상 기대
- BM25 성능 향상 기대 (같은 언어)

**테스트 계획**:
1. 한국어 QA 데이터만으로 테스트
2. 동일한 30개 어려운 질문으로 평가
3. 카테고리별 비교

### 7.2 한국어 + 영어 통합 테스트

**검색 흐름**:
```
질문 (한국어)
        ↓
[1단계] 한국어 데이터 검색 (RRF)
        ↓
[2단계] 영어 데이터 검색 (RRF)
        ↓
두 결과 통합 (가중치 조절)
        ↓
Top-5 최종 선정
```

**테스트 항목**:
- 통합 검색의 정확도
- 한국어/영어 결과 가중치 최적화
- 검색 속도 측정

### 7.3 최종 목표

| 지표 | 현재 (영어만) | 목표 (통합) |
|------|:------------:|:----------:|
| Top-5 Hit Rate | 93.3% | **95%+** |
| 증상-질병 카테고리 | 60.0% | **80%+** |

## 8. 요약

### 8.1 핵심 발견

1. **Cross-lingual 환경에서 시맨틱 > BM25**
   - multilingual-e5가 언어 장벽 극복
   - 시맨틱 90.0% > BM25 86.7%

2. **RRF 하이브리드가 가장 효과적**
   - RRF 93.3% = 시맨틱 + BM25 상호 보완
   - 시맨틱 대비 +3.3%p 향상

3. **증상-질병 질문은 여전히 어려움**
   - 둘 다 60%로 낮음
   - 구어체→의학용어 추론 한계
   - 한국어 QA 데이터 추가로 개선 기대

4. **고유명사(약물/음식)는 BM25가 강점**
   - "시금치", "자몽" 등 키워드 매칭
   - RRF에서 BM25가 시맨틱 보완

### 8.2 현재 한계

- 영어 데이터만 테스트한 결과
- 실제 서비스는 한국어 우선 검색 필요
- 한국어 QA 데이터 임베딩 후 재테스트 필요

### 8.3 다음 단계

1. ⏳ 한국어 QA 데이터 임베딩
2. ⏳ 한국어 데이터만 성능 테스트
3. ⏳ 한국어 + 영어 통합 성능 테스트
4. ⏳ 최종 검색 파이프라인 확정
