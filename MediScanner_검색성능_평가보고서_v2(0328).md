# MediScanner 검색 성능 평가 보고서 (v2)

## 📚 1. 배경: MedRAG 논문의 검색 방식 비교

### MedRAG 논문 개요
- **논문명:** "Benchmarking Retrieval-Augmented Generation for Medicine"
- **발표:** ACL 2024 Findings (2024년 8월)
- **저자:** Xiong et al. (University of Virginia, NIH)
- **핵심:** 의료 QA를 위한 RAG 시스템의 체계적 벤치마크

### MedRAG에서 비교한 3가지 검색 방식

| 검색 방식 | 설명 |
|-----------|------|
| **시맨틱 (Semantic)** | 벡터 임베딩 기반 의미 유사도 검색 (MedCPT, Contriever 등) |
| **BM25** | 키워드 빈도 기반 전통적 검색 알고리즘 (Best Matching 25) |
| **RRF (Reciprocal Rank Fusion)** | 시맨틱 + BM25 결과를 융합하는 하이브리드 방식 |

### MedRAG 논문 결과 (GPT-3.5 + MedCorp 코퍼스)

| 검색 방식 | MIRAGE 평균 정확도 |
|-----------|:------------------:|
| BM25 | 70.05% |
| MedCPT (시맨틱) | 70.06% |
| **RRF-4 (융합)** | **71.57%** ✅ |

> **논문 결론:** "The fusion of retrievers can provide robust performance... RRF-4 brings a larger improvement with a state-of-the-art average score of **71.57%**"

## 🔬 2. MediScanner의 동일 방식 비교 실험

### 실험 설계
MedRAG 논문과 동일한 3가지 검색 방식을 MediScanner에 적용하여 비교

| 비교 요소 | MedRAG 논문 | MediScanner |
|-----------|-------------|-------------|
| 임베딩 모델 | MedCPT (109M 파라미터) | E5-large (1024차원) |
| 코퍼스 | MedCorp (54.2M snippets, 영어) | ChromaDB (676,570 청크, 한국어) |
| 평가 데이터 | MIRAGE 벤치마크 (7,663개) | 자체 테스트셋 (200개) |
| 언어 | 영어 | 한국어 |

### MediScanner 테스트셋 구성

| 구분 | 개수 | 특징 | 예시 |
|------|:----:|------|------|
| 기본 질문 | 100개 | 직접적 의학 용어 사용 | "고혈압 증상은 무엇인가요?" |
| 어려운 질문 | 100개 | 구어체, 간접 표현, 추론 필요 | "머리가 어지러운데 고혈압에 안좋아?" |
| **전체** | **200개** | 7개 카테고리 균등 분포 | - |

## 📊 3. Top-K 개념 및 비교

### 3.1 Top-K란?

**Top-K**는 검색 시스템에서 가장 관련성이 높은 상위 K개의 문서를 반환하는 것을 의미합니다.

```
사용자 질문: "고혈압 증상이 뭐야?"
            ↓
     676,570개 문서에서 검색
            ↓
┌─────────────────────────────────────┐
│  Top-3: 상위 3개 문서만 반환        │
│  Top-5: 상위 5개 문서만 반환        │
│  Top-10: 상위 10개 문서만 반환      │
└─────────────────────────────────────┘
            ↓
     GPT-4o-mini가 답변 생성
```

### 3.2 Top-K 값에 따른 트레이드오프

| 항목 | Top-3 | Top-5 | Top-10 |
|------|:-----:|:-----:|:------:|
| 검색 정확도 | 낮음 | 중간 | 높음 |
| LLM 토큰 비용 | 적음 | 중간 | 많음 |
| 응답 속도 | 빠름 | 중간 | 느림 |
| 컨텍스트 길이 | 짧음 | 중간 | 김 |

### 3.3 Top-5 vs Top-10 정확도 비교 (E5-large)

| 질문 유형 | 검색 방식 | Top-5 | Top-10 | 차이 |
|----------|-----------|:-----:|:------:|:----:|
| **기본 (100개)** | 시맨틱 | 100.0% | 100.0% | 0%p |
| | BM25 | 99.0% | 100.0% | +1%p |
| | RRF | 100.0% | 100.0% | 0%p |
| **어려운 (100개)** | 시맨틱 | 89.0% | 94.0% | +5%p |
| | BM25 | 85.0% | 94.0% | +9%p |
| | RRF | 99.0% | 100.0% | +1%p |
| **전체 (200개)** | 시맨틱 | 94.5% | 97.0% | +2.5%p |
| | BM25 | 92.0% | 97.0% | +5%p |
| | RRF | 99.5% | 100.0% | +0.5%p |

### 3.4 Top-K 선택 결론: Top-5 채택

```
┌────────────────────────────────────────────────────────────────┐
│  📌 Top-5 선택 이유                                            │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  1️⃣ 정확도 측면                                                │
│     • Top-5 RRF: 99.5% vs Top-10 RRF: 100.0%                  │
│     • 차이: 0.5%p → 실질적 차이 미미                           │
│                                                                │
│  2️⃣ 비용 측면                                                  │
│     • Top-5: 약 2,500 토큰 (5문서 × 500자)                     │
│     • Top-10: 약 5,000 토큰 (10문서 × 500자)                   │
│     • GPT-4o-mini 비용 2배 절감                                │
│                                                                │
│  3️⃣ 속도 측면                                                  │
│     • Top-5: 더 적은 컨텍스트 → 빠른 응답                      │
│     • 노인 사용자 대상 → 응답 속도 중요                        │
│                                                                │
│  ✅ 결론: Top-5로 충분한 정확도(99.5%) 달성                     │
│           비용/속도 효율성을 위해 Top-5 채택                   │
└────────────────────────────────────────────────────────────────┘
```
